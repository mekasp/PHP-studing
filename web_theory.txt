Сервер - это компьютер,который предназначен для общего пользования
Веб-сервер - это сервер,который имеет возможность общаться с сетью интернет,с помощью програм,таких как :Apache/Ngnix.Сервер принимающий HTTP запросы и выдающий HTTP ответы

Модель OSI - 7ми уровневая сетевая модель.

    Уровни модели OSI:

        7 - прикладной уровень,обеспечивае взаимодействие сети и пользовотеля.
        Протокол: HTTP

        6 - уровень представления,служит для кодирования и упаковки данных
        Протокол: ASCII

        5 - сеансовый уровень,кординирует связь
        Протокол: L2TP

        4 - транспортный уровень,контролирует очередность прохождения компанентов
        Протокол: TCP,UDP

        3 - сетевой,адресация и доставка сообщений
        Протокол: IPv6

        2 - канальный,предназначен для передачи узам,находящийся в той же локальной сети
        Протокол: Ethernet

        1 - физический,физическая и электрическая среда для передачи данных
        Пример: кабель,витая пара,USB кабель,оптоволоконный кабель,радиоканал

Mодель TCP/IP

Модель TCP/IP - сетевая одель,имеющая 4 уровня,вместо 7ми представленных в модели OSI

    Уровни модели TCP/IP:

        4 - прикладной уровень

        3 - транспортный уровень

        2 - сетевой уровень

        1 - канальный


Пример отправки http запроса на youtube.com 0по модели TCP/IP:

    1. Формируется http запрос,устанавливается связь с интеретом
    2. По протоколу TCP отправляются пакеты
    3. На сетевом уровне определяется путь отправки пакетов
    4. На канальном уровне биты отправляютя другому хосту
    5. Хост дикапсулирует данные
    6. Хост формирует ответ и отправляет его назад таким же способом 

HTTP(Hyper Text Transfer Protocol) - протокол прикладного уровня сетевой модели(7ой уровень)

http содержит
    1. заголовки вопросов и ответов - несут информативную цель и указвыют какой тип контента нужно передать
    2. методы отправки запроса :
        GET
        HEAD
        POST
        PUT
        PATCH
        DELETE
        OPTIONS
    3. коды ответа:
        100 - 199  иформационные
        200 - 299  успешные
        300 - 399  перенаправления
        400 - 499  клиентские ошибки
        500 - 599  серверные ошибки
    4. тело запроса и ответа
